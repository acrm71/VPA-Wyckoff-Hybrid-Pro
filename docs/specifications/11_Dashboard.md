# 1. Purpose

The Dashboard is the presentation layer of the indicator.

It shall display analytical outputs generated by upstream engines without performing analytical calculations itself.

The Dashboard shall:

- Summarise current market state.
- Present signal readiness.
- Display module health.
- Show confidence values.
- Highlight conflicts.
- Expose important diagnostics.
- Remain readable across all supported timeframes.

---

# 2. Responsibilities

The Dashboard shall:

- Consume published outputs.
- Never calculate Trend.
- Never calculate BOS.
- Never classify Wyckoff.
- Never interpret VPA.
- Never calculate scores.

It is a renderer only.

---

# 3. Design Principles

## Separation of Concerns

The Dashboard must remain completely independent from analysis.

If Dashboard code is removed, the analytical engine shall continue functioning unchanged.

---

## One Source of Truth

Every displayed value shall originate from exactly one upstream engine.

Examples:

Trend State
→ Trend Engine

Institutional Bias
→ Institutional Engine

Signal
→ Signal Scoring Engine

---

## Low Cognitive Load

The Dashboard should answer:

1. What is happening?

2. Why?

3. Should I care?

without requiring the user to interpret dozens of labels.

---

## Progressive Disclosure

Important information should appear first.

Diagnostic information should appear only when enabled.

Required modes:

Minimal space.

- Institutional Bias

---

### Standard

Production default.

Displays:

- Trend
- Structure
- Liquidity
- VPA
- Wyckoff
- Institutional
- Signal
- Confidence

---

### Expanded

Displays every module plus diagnostics.

Intended for research.

---

### Debug

Displays:

- Internal scores
- Evidence counts
- Conflict
- Module availability
- Processing times
- Lifecycle states

Never enabled by default.

# 29. Testing

Verify:

Every module updates correctly.

Unavailable modules display N/A.

No stale values remain.

Debug mode reflects internal values.

Table remains within TradingView limits.

Theme switching works.

Position changes correctly.

No repainting.

---

# 30. Acceptance Criteria

Dashboard is complete when:

- No calculations occur inside Dashboard.
- One table is maintained.
- All analytical modules display correctly.
- Colours remain consistent.
- Themes function correctly.
- Compact, Standard, Expanded, and Debug modes function correctly.
- Object limits are respected.
- Performance impact remains negligible.
- Dashboard updates only after analytical pipeline completion.
